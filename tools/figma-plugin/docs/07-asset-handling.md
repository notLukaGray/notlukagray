# Asset Handling

The plugin exports two binary asset types: PNG (images and video posters) and SVG (vectors inlined as strings). Videos are not exported — Figma has no video data. All binary assets accumulate in `ctx.assets` during conversion and are written to the `assets/` folder in the ZIP.

---

## Asset key system

All asset filenames are generated by `buildAssetKey` in `src/utils/asset-key.ts`. This function:

1. Sanitizes the raw layer name: lowercased, spaces → hyphens, only alphanumeric/`/`/`-`/`_`/`.` allowed.
2. Appends `.png` (or a specified extension) if no extension is present.
3. Prepends the per-frame CDN prefix set in the plugin UI (e.g. `work/my-project/`).
4. Checks against `ctx.usedAssetKeys` and appends `-2`/`-3`/... on collision.
5. Returns both `cdnKey` (JSON reference, e.g. `hero/banner.png`) and `filename` (ZIP path, e.g. `assets/hero/banner.png`).

### Layer name → asset key examples

| Layer name    | CDN prefix    | cdnKey                      | ZIP path                           |
| ------------- | ------------- | --------------------------- | ---------------------------------- |
| `hero`        | —             | `hero.png`                  | `assets/hero.png`                  |
| `hero/banner` | —             | `hero/banner.png`           | `assets/hero/banner.png`           |
| `work/cover`  | `case-study/` | `case-study/work/cover.png` | `assets/case-study/work/cover.png` |
| `Frame 42`    | —             | `frame-42.png`              | `assets/frame-42.png`              |

The CDN prefix is set per-frame in the plugin UI (persisted in `localStorage`). It prepends to all asset keys generated during that frame's conversion.

## Images

Defined in `src/converters/image.ts`.

### Image fill path (preferred)

When a node has an `IMAGE` fill, the raw image bytes are retrieved via `figma.getImageByHash`:

```ts
const img = figma.getImageByHash(imageFill.hash);
const bytes = await img.getBytesAsync();
const assetKey = buildAssetKey(node.name, ctx);
ctx.assets.push({ filename: assetKey.filename, data: new Uint8Array(bytes) });
// → elementImage.src = assetKey.cdnKey
```

This preserves the original image data without re-encoding. `objectFit` is derived from the fill's `scaleMode`, and `objectPosition` is derived from `imageTransform` when meaningfully off-center.

### Rasterise path (fallback)

When no `IMAGE` fill is found (e.g. node annotated with `[pb: type=image]` but has no fill, or a complex layered node), the entire node is rasterised:

```ts
const bytes = await node.exportAsync({ format: "PNG" });
const assetKey = buildAssetKey(node.name, ctx);
```

`objectFit` is not set for rasterised exports.

### Aspect ratio

Computed from node dimensions using GCD simplification. Only emitted when both parts of the simplified fraction are ≤ 50:

```
1920 × 1080  →  "16/9"   (emitted)
800  × 600   →  "4/3"    (emitted)
317  × 218   →  undefined (too complex — not emitted)
```

### Object position

Derived from the `imageTransform` affine matrix in UV space. Only emitted when the position is more than 5% away from center (50%/50%) on either axis. For `CROP` scale mode, a warning is added noting the position is approximate.

Override with `[pb: objectPosition=top]` or `[pb: objectPosition=50% 30%]`.

### Background image fills on sections

Image fills on top-level section frames are exported separately via `exportImageFillAsset`. The key is derived from `{frameName}/bg`:

```
Frame named "hero"  →  cdnKey: "hero/bg.png"  →  ZIP: "assets/hero/bg.png"
```

The CDN key is stored as `bgImage` on the section block (not `fill`).

## SVGs

Defined in `src/converters/vector.ts`.

```ts
const markup = await node.exportAsync({ format: "SVG_STRING" });
// → elementSVG.markup = markup
```

SVGs are exported as strings and inlined directly in `elementSVG.markup`. No asset file is written to the ZIP. This applies to all vector-like node types: `VECTOR`, `BOOLEAN_OPERATION`, `STAR`, `POLYGON`, `LINE`, `ELLIPSE`, and plain `RECTANGLE` (no image fill).

---

## Videos

Videos are **not exported** — Figma has no video data to export. The plugin derives a CDN key from the layer name and emits that as `src`.

### src resolution priority

1. `[pb: src=path/to/video.mp4]` annotation — used as-is.
2. Component property named `src` with a video file extension — INSTANCE nodes only.
3. Derived from the clean layer name: `buildAssetKey(cleanName, ctx, ".mp4")`. Layer `work/project/intro-reel` → `work/project/intro-reel.mp4`. A warning is logged to verify the path.

### poster resolution priority

1. `[pb: poster=...]` annotation — used as-is, treated as a CDN key, no export needed.
2. Component property named `poster`.
3. The node's visual is exported as a PNG raster. Key: `{cleanName}/poster.png`. For layer `intro-reel`, the poster is `intro-reel/poster.png` in the ZIP and referenced in JSON as `intro-reel/poster.png`.

If poster export fails (e.g. empty node), a warning is logged and `poster` is omitted.

## ZIP structure

```
page-builder-export.zip
  pages/
    case-study.json
  presets/
    hero-dark.json
  modals/
    contact-form.json
  modules/
    feature-card.json
  globals.json
  assets/
    hero/banner.png
    hero/bg.png
    hero-reel/poster.png
    logo-mark.svg.png      ← SVG rasterised (inline in JSON, not here)
  export-notes.txt
```

`src` paths in JSON reference the CDN key directly: `"src": "hero/banner.png"`. After uploading `assets/` to a CDN, these paths become the CDN URL path component.

`assetCounter` is global across all frames in a single export run. Asset keys are collision-detected via `ctx.usedAssetKeys`.

## After export: placing assets

Asset keys in JSON are the CDN path (without the `assets/` prefix from the ZIP). After unzipping:

**CDN upload (recommended)**:

```
Upload assets/ to CDN root
hero/banner.png → https://cdn.example.com/hero/banner.png
```

**Local serving**:

```
assets/hero/banner.png → /public/assets/hero/banner.png
```

Update JSON `src` values if the CDN base URL differs from what the plugin generated.

Always validate after placing files:

```bash
npx tsx scripts/validate-pages.ts
```
